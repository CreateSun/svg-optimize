# SVG 图标导入与预设扩展 PRD

## AI 速读卡
产品一句话：为 SVG 编辑器增加更多预设与外部图标库导入能力，让设计师和前端能从找图标到进入编辑只走一条最短路径。
核心循环：打开导入面板 -> 搜索或粘贴图标源 -> 预览并导入 -> 在代码编辑器中修改 -> 导出 SVG 或 PNG。
目标平台：桌面优先 Web 应用，移动端只保证可浏览与基础导入，不承诺完整编辑体验。
硬约束：导入结果必须落成可编辑 SVG 字符串；P0 不依赖登录；外链失败必须可恢复；不得误导用户认为已接入受限版权资源。
推荐默认：先做更多内置预设、Lucide 搜索导入、iconfont symbol 链接导入；Font Awesome 仅规划 Free 路线，不进首发。
发挥空间：导入面板视觉、搜索交互动效、最近导入、收藏夹、批量对比、空状态文案可自由增强。
P0 验收：用户能从预设、Lucide、iconfont 三个来源成功导入 SVG，并立即进入现有编辑器继续编辑和导出。
最容易翻车：把外部平台接成“黑盒跳转”而不是“导入为本地可编辑 SVG”，以及未处理链接失效、跨域失败、图标格式不标准。
超预期机会：最近导入、变体对比、导入后自动格式化、图标来源标记与二次检索。

## 第一章：产品概述
SVG 图标导入与预设扩展是一套创作者工具增强能力，让设计师、前端开发与运营同学能够快速导入高质量 SVG 图标并继续编辑，而无需在多个图标站点、下载目录与代码编辑器之间来回切换。

### 1.1 差异化对比表

| 功能 | 竞品 | 本产品 | 实现方式 |
|---|---|---|---|
| 从图标站进入编辑 | 多数图标站只提供复制或下载，编辑发生在站外 | 在同一工作流内完成搜索、导入、编辑、导出 | 在 SVG 编辑器内嵌入导入面板与标准化解析层 |
| iconfont 项目级导入 | 常见做法是复制项目链接后手工找 symbol 名称 | 粘贴项目链接后直接列出图标并点选导入 | 解析 iconfont symbol 脚本，提取 symbol 节点并转换为独立 SVG |
| 图标导入后的二次编辑 | 大多平台强调分发，不强调继续改代码 | 导入完成即进入代码编辑器，保留格式化、补全、PNG 导出 | 统一把导入结果落成字符串并写入现有编辑状态 |
| 预设与外部图标并存 | 常见是模板站或图标站二选一 | 预设、外部图标、上传文件都走同一编辑面板 | 建立统一导入入口与来源标记 |

### 1.2 三类用户画像

角色一：前端开发
核心目标：快速找到一个合适图标并微调大小、描边、颜色、语义属性。
对现有工具最大的不满：在多个站点复制 SVG 后还要手工清洗、补 viewBox、删无用属性。
让他们愿意切换的那一个功能：导入后立即是可编辑 SVG 源码，而不是图片预览或下载文件。

角色二：产品设计师
核心目标：先挑一个接近的图标，再快速做局部结构调整用于界面方案。
对现有工具最大的不满：图标平台搜索强，但导入后无法继续深改或做版本对比。
让他们愿意切换的那一个功能：从图标库点选进入编辑，并保留多版本候选。

角色三：内容运营或低代码同学
核心目标：不依赖设计软件，快速产出可下载的 SVG 与 PNG 图标素材。
对现有工具最大的不满：平台术语多、代码门槛高、失败后不知如何恢复。
让他们愿意切换的那一个功能：链接导入成功后直接可见预览，并能一键导出 PNG。

### 1.3 可行性边界

| 在范围内及原因 | 明确排除在外及原因 |
|---|---|
| 内置预设扩容，原因是完全可本地维护且无外部依赖 | 首发支持所有商业图标平台账号体系，原因是授权与接口复杂度高 |
| Lucide 导入，原因是适合做开源友好的静态索引型导入 | 首发支持受登录保护的 Pro 图标资源，原因是权限、计费和法务边界未解决 |
| iconfont symbol 链接导入，原因是符合用户直接粘贴链接的一键导入心智 | 通过抓取整站搜索结果绕过平台产品流程，原因是稳定性与合规风险高 |
| 导入后继续编辑并导出 SVG 或 PNG，原因是现有编辑器链路已存在 | 多文件项目管理、团队共享、云端同步，原因是会显著扩大产品边界 |
| 桌面 Web 完整体验，原因是当前编辑器已经是桌面优先布局 | 移动端完整代码编辑体验，原因是行号、补全与预览在小屏上可用性不足 |

### 1.4 约束分层

| 硬约束 | 推荐默认 | 发挥空间 |
|---|---|---|
| 所有导入结果必须变成标准 SVG 字符串并进入编辑器主态 | 导入完成后自动执行一次格式化与空标签自闭合标准化 | 导入卡片样式、过渡动画、来源徽标、搜索空状态文案可优化 |
| 用户不登录也能使用 P0 全链路 | 默认保留来源标签，便于用户知道图标来自预设、Lucide 或 iconfont | 可增加最近导入、收藏、相似图标推荐 |
| 失败时必须给出可恢复操作，不允许静默失败 | 默认把 Lucide 做成内置索引搜索，不依赖运行时访问第三方官网 | 可增加批量预览、导入前对比、颜色主题建议 |
| 不承诺未验证或受限的图标平台可用性 | 默认首发只宣称支持 Lucide 与 iconfont symbol 链接 | 可在后续接 Font Awesome Free 与更多来源 |
| 导入能力不能破坏现有上传、编辑、导出链路 | 默认保留本地优先，不新增账号与云端存储 | 可补充引导式上手与快捷键 |

## 第四章：超越竞品的差异化功能

### 第 4.1 节 功能名称：从 iconfont 项目链接直接进入 SVG 编辑

一，竞品为何没有这个功能
基于公开信息的推断：大多数图标平台的主目标是分发图标，而不是承担后续编辑器角色；项目链接更偏向团队资源管理，不会优先做成“粘贴后再进入深编辑”的一体化体验。

二，本产品如何实现
推荐做法是前端先校验链接是否为 symbol 项目链接，再拉取脚本文本，解析其中 symbol 节点并在本地生成候选 SVG。替代方案是服务端代理抓取，但会引入部署复杂度和潜在合规风险，因此 P0 先做纯前端拉取，失败时清晰提示。

三，交互流程

粘贴 iconfont symbol 链接
    |
    v
解析成功
    |
    v
列出图标
    |
    v
点选某个 symbol
    |
    v
转换为独立 SVG
    |
    v
写入编辑器并自动格式化

四，风险与应对
风险一：跨域或网络失败。应对：给出重试、示例格式、手动粘贴原始 SVG 备选方案。
风险二：脚本文本结构变更。应对：解析器按 symbol 节点容错匹配，并记录失败样本。
风险三：用户误用 font-class 链接。应对：输入框下明确只支持 symbol 链接。

### 第 4.2 节 功能名称：图标导入后立即进入源码级编辑

一，竞品为何没有这个功能
很多平台的产品边界停在搜索、预览、复制和下载；一旦进入代码级编辑，就需要处理结构规范化、预览同步、导出链路与错误恢复，这不是其核心商业路径。

二，本产品如何实现
无论来源是什么，都通过统一标准化层输出 SVG 字符串并写入现有编辑器状态。这样用户不需要学习新的“导入专用编辑模式”，产品认知成本最低。

三，交互流程

选择图标来源
    |
    v
点导入
    |
    v
SVG 规范化
    |
    +-- 成功 --> 编辑器展示源码 --> 用户继续改
    |
    +-- 失败 --> 错误提示 --> 保留原内容

四，风险与应对
风险一：导入图标覆盖用户当前内容。应对：导入前若已有未保存修改，则二次确认。
风险二：格式化破坏个别特殊标签布局。应对：保留最小可逆规则，不做语义重写。

### 第 4.3 节 功能名称：统一来源抽屉与最近导入

一，竞品为何没有这个功能
此处未验证。基于创作者工具的一般模式推断，图标平台通常只处理自己的内容池，不会把内置预设、第三方图标和本地上传统一到同一入口。

二，本产品如何实现
所有来源放入同一抽屉结构，并记录最近导入。技术上只需保留一层来源元信息，不增加复杂数据模型。

三，交互流程

四，风险与应对
风险一：抽屉信息过载。应对：P0 保持三到四个来源，不做过多运营入口。
风险二：最近导入污染本地存储。应对：仅保存最近 20 条。

## 第五章：数据模型

```json
{
  "version": "1.0", // 必填，导入模块数据版本
  "importSource": {
    "type": "lucide", // 必填，可选值：preset、lucide、iconfont、url
    "label": "Lucide", // 必填，来源展示名
    "origin": "local-index", // 必填，来源定位信息；默认值: local-index
    "rawRef": "search", // 选填，原始引用值，如图标名、链接或 symbol id
    "importedAt": 1750000000000 // 必填，毫秒时间戳
  },
  "iconCandidate": {
    "id": "search", // 必填，候选图标唯一标识
    "name": "search", // 必填，用户可读名称
    "tags": "ui,navigation", // 选填，搜索辅助标签，逗号分隔
    "svg": "<svg viewBox=\"0 0 24 24\">...</svg>", // 必填，标准 SVG 字符串
    "previewable": true // 必填，默认值: true
  },
  "recentImports": {
    "items": "最多 20 条", // 必填，最近导入记录集合说明
    "storageKey": "svg-edit-recent-imports", // 必填，本地存储键名
    "maxCount": 20 // 必填，默认值: 20
  }
}
```

```json
{
  "version": "1.0", // 必填，iconfont 解析结果版本
  "iconfontProject": {
    "projectUrl": "//at.alicdn.com/t/c/font_xxx.js", // 必填，用户输入链接
    "fetchStatus": "success", // 必填，可选值：idle、loading、success、error
    "errorCode": "", // 选填，默认值: 空字符串
    "symbols": "symbol 列表", // 必填，解析后的 symbol 集合说明
    "parsedAt": 1750000000000 // 选填，解析成功时间
  }
}
```

核心设计决策：数据模型故意保持轻量，不引入账号、项目、云端资产表。P0 的核心是导入并进入编辑，而不是管理大量素材资产，因此只保留来源元信息、候选图标和最近导入即可。刻意排除了团队库、收藏夹同步和权限模型。

## 第六章：技术架构

应用层
+-- 导入面板层：承接来源切换、搜索、输入、选择
+-- 导入解析层：把预设、Lucide、iconfont 链接统一转成候选 SVG
+-- 标准化层：补根节点、格式化、自闭合、错误校验
+-- 编辑器状态层：写入 svgSource、状态提示、最近导入
+-- 预览与导出层：复用现有预览、PNG 导出、截图能力
+-- 本地持久层：保存最近导入与面板偏好

| 库名 | 用途 | 为何优于替代方案 | 大致包体积 |
|---|---|---|---|
| @uiw/react-codemirror | SVG 代码编辑器 | 已在当前项目内落地，避免再引入第二套编辑器 | 未知 |
| @codemirror 系列 | 行号、高亮、补全 | 已接入且可复用现有编辑体验 | 未知 |
| lucide-static 或等价静态资源方案 | Lucide 图标索引与 SVG 内容 | 本地索引型接入简单、稳定、开源友好 | 未知 |
| 原生 DOMParser 与 XMLSerializer | SVG 解析和标准化 | 浏览器内置，避免额外解析依赖 | 0 KB |

最大架构风险：iconfont 链接解析高度依赖远端脚本文本结构与可访问性，一旦链接跨域受限或结构变化，导入会不稳定。应对策略是把 iconfont 解析器独立成单一模块，失败时不影响其他来源，并为后续服务端代理预留接口边界。

可替换技术原则：
推荐沿用现有 React 加 CodeMirror 方案；如果项目已有 Monaco，也可改为 Monaco 实现导入后的编辑体验；Lucide 索引可用本地 JSON、静态资源包或构建期脚本生成。不可变的是：所有来源都必须统一落成可编辑 SVG 字符串，且失败时不能覆盖用户当前内容。

### 7.3 空状态

预设页空状态：显示“没有找到对应预设”，主按钮为“切换到 Lucide 搜索”。
Lucide 空状态：显示推荐关键词，如 search、user、settings，主按钮为“清空关键词”。
iconfont 空状态：显示示例 symbol 链接格式，主按钮为“粘贴链接后解析”。
最近导入空状态：显示“你还没有导入过图标”，主按钮为“打开导入面板”。

### 7.4 错误状态

| 触发条件 | 用户可见的提示信息 | 恢复操作 |
|---|---|---|
| iconfont 链接格式错误 | 仅支持 iconfont symbol 项目链接，请更换后重试 | 编辑链接并再次解析 |
| iconfont 拉取失败 | 无法访问该链接，请检查网络或链接公开性 | 重试或改用手动粘贴 SVG |
| iconfont 解析后无 symbol | 该链接未解析出可导入图标 | 更换链接 |
| Lucide 索引加载失败 | Lucide 图标索引加载失败 | 刷新页面并重试 |
| 导入 SVG 非法 | 导入内容不是有效 SVG，已保留当前编辑内容 | 返回来源重试或手动粘贴 |

### 7.5 加载状态

Lucide 本地索引过滤：小于 80ms 不展示 loading；超过 80ms 显示结果骨架屏。
iconfont 链接解析：立即显示解析中状态；超过 300ms 显示“正在解析链接”文案。
导入标准化写入：小于 100ms 不额外显示遮罩；超过 100ms 在状态栏显示“正在导入”。
PNG 与 Screenshot 导出：立即禁用导出按钮；超过 200ms 显示轻量进度提示。

## 第八章：导出与输出系统

### 8.1 支持的输出格式

| 格式 | 使用场景 | 质量选项 | 备注 |
|---|---|---|---|
| SVG | 继续开发、设计交付、再次编辑 | 无损 | 导入后编辑结果的主输出 |
| PNG | 运营素材、文档插图、低代码上传 | 默认 2 倍导出 | 透明背景 |

SVG 输出：单文件，文件名默认 `svg-edit-export.svg`。
PNG 输出：单文件，文件名默认 `svg-edit-export.png`。
Screenshot 输出：单文件，文件名默认 `svg-edit-screenshot.png`。
来源元信息不写入导出文件，仅用于产品内显示和最近导入。

### 8.3 输出流程

    |

### 8.4 输出失败兜底

若 PNG 或 Screenshot 导出失败，需提示用户先下载 SVG；不得让用户误以为内容已丢失。

## 第十一章：开发者交接说明

### a) 你要先做什么

你先抽象出统一导入面板状态，不要一开始把逻辑塞进现有预设侧栏。先把来源切换、搜索词、候选列表、导入动作和最近导入抽成独立状态层，再把预设、Lucide、iconfont 三个来源接进去。

### b) 你不能擅自改什么

你不能把“导入”做成只复制 SVG 文本而不进入编辑器的半链路。
你不能为了接外部图标源而破坏现有编辑器、格式化、PNG 导出和 Screenshot。
你不能在未验证授权边界前对外宣称已支持所有图标平台。

### c) 你可以自由发挥什么

你可以优化导入抽屉的视觉层次、卡片布局、最近导入展现方式、搜索交互细节和动画。
你可以决定最近导入放在侧栏还是抽屉顶部，只要主路径不变。
你可以根据代码结构拆成新组件，但要保证来源模块职责清晰。

### d) 已知的未知项

此处未解决：iconfont 某些项目链接是否会因为公开性设置导致前端无法稳定访问。
此处未解决：Lucide 首发是直接引入静态包还是构建期生成索引，需结合当前包体与构建方式决定。
未知：后续 Font Awesome Free 接入时，是否采用官方包还是通过用户提供 token 的方式更适合当前产品定位。

### e) 验收剧本

验收剧本 1
打开页面，点击导入面板，进入预设来源，点选任意预设。
可观察证据：编辑器内容被替换为标准 SVG，预览立即刷新，状态栏提示导入成功。

验收剧本 2
打开 Lucide 来源，输入 `search`，在结果中点选一个图标导入。
可观察证据：结果列表出现匹配项，导入后保留两空格缩进和空标签自闭合，SVG 可继续编辑。

验收剧本 3
进入 iconfont 来源，输入一条有效的 symbol 项目链接并解析，选择其中一个图标导入。
可观察证据：解析成功后出现图标列表，导入后编辑器中出现独立 SVG，而不是 `<use>` 片段。

验收剧本 4
在 iconfont 来源输入错误链接格式。
可观察证据：系统明确提示仅支持 symbol 链接，不覆盖当前编辑器内容，且用户可以继续修改输入并重试。

验收剧本 5
导入任意来源的图标后，执行 SVG 下载、PNG 导出和 Screenshot。
可观察证据：三个导出入口都可用，且文件内容与当前编辑结果一致。

### f) 首版完成定义

当用户可以从预设、Lucide、iconfont 三个来源导入 SVG，并在同一页面完成编辑、预览与导出，而且所有失败路径都有恢复动作时，就算首版完成。不要把账号、收藏同步、商业图库接入塞进首版。
